Java基础（一）：基本概念

  1. Java基础部分
  2. 面向对象部分
  3. 常用API部分
  4. 集合与IO部分

二、Windows常用快捷键

  • Windows徽标键+L键锁屏
  • Windows徽标键+E键启动“我的电脑”
  • Windows徽标键+D键快速显示桌面
  • Windows徽标键+R键执行“运行”命令
  • Ctrl+A 全选
  • Ctrl+C 复制
  • Ctrl+V 粘贴
  • Ctrl+X 剪切
  • Ctrl+Z 撤销
  • Ctrl+S 保存

三、常用DOS命令

  • d: 回车 盘符切换
  • dir(directory):列出当前目录下的文件以及文件夹
  • cd (change directory)改变指定目录(进入指定目录)
  • cd.. 退回到上一级目录
  • cd\ 退回到根目录
  • md (make directory) : 创建目录
  • rd (remove directory): 删除空目录
  • del (delete): 删除文件,删除一堆后缀名一样的文件.txt （是通配符,代表所有文件名）
  • cls : (clear screen)清屏
  • exit : 退出dos命令行

四、计算机存储单位及进制转换

  • bit(位)： 存放一位二进制数，即 0 或 1，最小的存储单位。
  • byte(字节)： 8个二进制位是一个字节，是最常用的单位。
  • 1KB=1024Byte
  • 1MB=1024KB
  • 1GB=1024MB
  • 1TB=1024GB
  • 1PB=1024TB
  • 二进制： 相当于电信号，只有开和关，即0和1，满2进1，用ob或者0B开头。
  • 十进制： 0-9 ，满10进1 。
  • 八进制： 0-7 ，满8进1，用0开头表示。
  • 十六进制： 0-9，A-F，满16进1，用0x或者0X开头表示。

五、Java的跨平台

  1. 什么是跨平台？
    跨平台即为开发出来的程序可以运行于所有平台。（平台==各种操作系统 跨==程序可以运行在各种操作系统）
  2. Java跨平台的原理？
    Java程序是运行在Java虚拟机（JVM）里的，而Java虚拟机是针对不同的平台（操作系统：Windows、Liunx、Mac等）开发出来并运行在各个平台上的，所以Java语言开发出来的程序可以跨平台运行。

六、JRE和JDK

  1. JRE(Java Runtime Environment):
    包括Java虚拟机(JVM: Java Virtual Machine)和Java程序所需的核心类库等，如果想要运行一个开发好的Java程序，计算机中只需要安装JRE即可。
  2. JDK(Java Development Kit):
    JDK是提供给Java开发人员使用的，其中包含了java的编译器、开发工具、打包工具等，也包括了JRE。所以安装了JDK，就不用再单独安装JRE了。

七、开发Java程序的三个步骤（初级）

  1. 编写程序源代码：
    用文本编辑器（EditPlus、NotePad++、记事本）编写Java程序源代码，后缀名改为.java。
  2. 将编写好的程序源代码编译为字节码文件：
    打开命令行，利用cd命令进入JDK的bin目录下，输入javac 源文件路径\源文件名.java，回车，会在源文件所在路径下生成 源文件名.class的字节码文件。
  3. 运行编译完成的程序（字节码文件）：
    将编译好的字节码文件复制到JDK的bin目录下，打开命令行，利用cd命令进入JDK的bin目录下，输入java 源文件名，回车，会在命令行显示程序的运行结果。

八、配置环境变量——path

  1. 什么是环境变量：
    环境变量是操作系统中的东西，是用来设置路径的，设置好路径后，该路径下的程序就可以在任何地方直接使用。
  2. 怎么配置环境变量：（找系统变量中的Path并编辑）
    a.Path中已经有了很多路径,需要把JDK的bin路径加入到Path中，这样javac，java等命令即可在任何地方使用（路径之间必须用英文分号隔开）。
    b.把JDK的根路径放到新建的JAVA_HOME中，把%JAVA_HOME%\bin加入到Path中（其中%%为取变量符号）。
  3. 配置好环境变量后怎么开发Java程序：
    用记事本编辑好程序源代码，打开命令行，利用cd命令进入源文件所在的路径，然后输入javac xxx.java，最后输入java xxx即可看到程序运行结果。

九、配置环境变量—classpath(现在已经不需要了)

  1. classpath用来配置字节码文件的路径,配置完成后可以在任何路径下直接使用字节码文件。
  2. 一般不需要配置classpath，因为在第二步编译源文件时,必须先进入到源文件所在的路径,而字节码文件默认也在这个路径下,因此配置classpath没有多大意义。

十、注释

  • 什么是注释：
    注释就是对某块代码进行解释说明的文字。
  • 注意：
    注释不能直接写到源文件中，否则编译失败。
  1. 单行注释（//）
    编译过程中会把单行注释后的文字忽略掉。
  2. 多行注释（/ /）
    编译过程中会把多行注释中的文字忽略掉。

十一、转义符

  • 换行
    a.\n：输出一个换行，必须放到””内。
    b.System.out.println(); 会自动在最后输出一个换行。
  • 制表符
    在Java中,”\t”表示一个制表符，可以保证制表符后边的内容对齐。
  • 双引号
    在Java中,”\””表示一个双引号。
